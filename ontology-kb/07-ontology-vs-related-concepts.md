# ontology 与相关概念对照

## ontology

- 定义领域概念、关系和约束
- 强调语义和可推理性

## taxonomy

- 主要是分类层级
- 关注“谁是谁的子类”

## schema

- 主要是数据结构定义
- 关注“数据怎么存”

## knowledge graph

- 主要是把实体和关系连接成图
- ontology 常作为知识图谱的语义层

## data model

- 主要是系统实现层的数据设计
- 更偏工程结构

## glossary

- 主要是术语解释
- 不一定包含完整关系和规则

## 什么时候用什么

- 只想做层级分类，用 taxonomy
- 只想设计数据库，用 schema
- 想表达领域语义和规则，用 ontology
- 想把知识和数据连起来，用 knowledge graph
